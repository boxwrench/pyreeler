## Running the Demo

```bash
cd experimental/experiments/parameter-sequence-demo
python main.py
```

**Requirements:**
- numpy, pillow, scipy
- FFmpeg (for video encoding)

**Output:**
```
output/
├── demo.mp4              # Final film
├── audio.wav             # Generated audio
├── sequences/
│   ├── lorenz.json       # Parameter sequences (editable)
│   ├── rossler.json
│   ├── bell.json
│   └── drone.json
└── frames/               # Individual PNG frames
```

## Modifying Sequences

Edit the JSON files in `output/sequences/` and re-run:

```python
from tools.parameter_sequence import ParameterSequence

# Load existing sequence
seq = ParameterSequence.load("output/sequences/lorenz.json")

# Modify
seq.record(180, 'trail_length', 800)  # Add keyframe

# Save
seq.save("output/sequences/lorenz.json")

# Re-render (only video affected by this change)
python main.py
```

Or edit directly in `main.py` in the `create_sequences()` function.

## Performance Notes

| Phase | Time | Bottleneck |
|-------|------|------------|
| Precomputation | ~10-20s | Lorenz trajectory generation |
| Video render | ~20-40s | Frame rasterization |
| Audio render | ~2-5s | FM synthesis |
| FFmpeg encode | ~5s | Video compression |
| **Total** | **~40-70s** | - |

**Optimization:** If render is too slow, reduce `WIDTH`, `HEIGHT`, or `TOTAL_FRAMES` in `main.py`.

## Key Insight

The same `ParameterSequence` class drives:
- Visual: trail length, particle count, camera zoom
- Audio: modulation index, carrier frequency, harmonic ratio

This cross-domain parameter automation means a single sequence file can define a complete audio-visual experience. Edit the JSON, share it, version control it.
